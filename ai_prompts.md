# AI Prompts 紀錄

本文件記錄在審計過程中使用的每一個 AI prompt 及其回應摘要。

---

## Prompt 1：初步探索資料

**Prompt（由 AI 自動執行）**：
```
讀取 CSV 前 3 行，確認欄位結構與編碼
```

**AI 回應摘要**：
- 確認檔案有 5,974 行（含標頭），共 15 個欄位
- 欄位包含：序號、縣市及鄉鎮市區、村里、地址、經度、緯度、名稱、收容村里、收容人數、災害類別、管理人姓名/電話、室內/室外/弱者安置
- 編碼為 UTF-8 with BOM

---

## Prompt 2：空值與座標全面檢查

**Prompt（由 AI 自動執行）**：
```python
# 各欄位空值統計
# 經緯度檢查：零值、超出台灣範圍(118~123, 21~27)、經緯度互換
```

**AI 回應摘要**：
- 發現 3 個欄位有空值：村里(209)、適用災害類別(184)、地址(3)
- 3 筆經度為 0（高雄市三個避難所）
- 2 筆經度偏小（117.x，南投、宜蘭）→ 超出台灣範圍
- 無經緯度互換的案例

**效果評估**：AI 成功找到明顯的數值異常，但空值檢查只涵蓋部分欄位。

---

## Prompt 3：收容人數、重複紀錄、序號檢查

**Prompt（由 AI 自動執行）**：
```python
# 收容人數：非數字、負數、超大值(>10000)
# 重複資料：名稱+地址完全相同
# 序號是否連續
```

**AI 回應摘要**：
- 8 筆收容人數超過 10,000（最大 34,000 為河濱公園）
- 15 組完全重複的紀錄
- 序號連續正確（1~5973）

**效果評估**：AI 正確抓到超大值和重複紀錄。序號檢查通過。

---

## Prompt 4：座標精度、座標重複、科學記號、電話異常

**Prompt（由 AI 自動執行）**：
```python
# 座標精度：小數位數<3的紀錄
# 座標重複：不同名稱卻相同座標
# 科學記號電話（Excel 污染）
```

**AI 回應摘要**：
- 78 筆低精度座標（小數<3位），其中 6 筆為純整數座標（如 120, 23）
- 211 組不同避難所共用相同座標
- 7 筆電話被 Excel 轉成科學記號（如 4.72E+11）
- 1 筆電話欄位填入中文「陳政治」

**效果評估**：這是最有價值的一輪。座標重複問題（211 組）是最大發現。科學記號問題也是 AI 主動找出的。

## Prompt 5：嚴重座標問題深入調查

**Prompt（由 AI 自動執行）**：
```python
# 逐一列出：經度=0、緯度整數截斷、經度<118
# 整數座標的具體紀錄
# 電話欄位填入純中文的案例
```

**AI 回應摘要**：
- 確認 3 筆經度為 0 都在高雄市
- 2 筆緯度被截斷為整數（26、27），在地圖上會跑到台灣以外
- 2 筆經度 117.x 分別在南投和宜蘭，推測應為 120.x 或 121.x
- 行 3817 電話欄填入「陳政治」確認為欄位填錯

---

## AI 的盲點與限制

1. **無法驗證地址正確性**：AI 能檢查格式，但無法確認「新竹縣某地址」是否真的在新竹縣
2. **無法判斷座標是否真的指向該建築物**：即使座標在合理範圍內，也可能偏移數百公尺
3. **收容人數合理性需要領域知識**：34,000 人的河濱公園到底合不合理，AI 無法確定
4. **科學記號無法還原**：`4.72E+11` 還原後是 472000000000，這不是有效電話號碼，原始資料已遺失
